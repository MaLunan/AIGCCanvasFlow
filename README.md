# AIGCCanvasFlow
无限画布编辑器


一、整体产品定位
AI 可视化节点式视频创作平台，核心：无限画布 + 可视化节点工作流 + 多 AI 模型调度 + 短剧 / 分镜自动化生产，完全对标 LibTV 核心体验。
二、技术栈选型（前后端 + AI + 存储 + 部署）
1. 前端（核心难点：无限画布、节点编辑）
框架：Vue3 + Vite + Pinia
画布引擎：vueFlow
视频处理：video.js + ffmpeg.wasm（前端剪辑、拼接、转码）
富文本 / 剧本编辑：Quill / Wangeditor
UI 组件：Element Plus / Arco Design
图形绘制：Fabric.js（宫格切图、图层、画布标注）
2. 后端
框架：Node.js NestJS 或 Java SpringBoot
数据库：
主库：MySQL（项目、用户、订单、权限）
缓存：Redis（任务队列、算力扣减、登录态）
消息队列：RabbitMQ / Redis MQ（AI 生成异步任务调度）
3. AI 模型层（中台化）
统一 AI 网关：统一接收前端请求、分发不同模型
接入模型：
图生图 / 文生图：通用绘画模型
视频生成：可灵 Kling、Seedance、万像
音频：AI 配音、TTS、音效、背景音乐
调用方式：HTTP/WS 接口，异步回调返回结果
4. 存储 & 资源
对象存储：阿里云 OSS / 腾讯 COS（图片、视频、音频、素材）
文件存储：分片上传、大文件断点续传
CDN：静态资源、成片视频加速分发
5. 部署 & 运维
容器：Docker + Docker Compose
服务编排：K8s（中后期）
服务器：云服务器 + 算力服务器分离
内容审核：第三方内容安全接口（图片 / 视频 / 文本鉴黄、敏感内容）
三、系统整体架构分层
前端应用层：画布编辑器、项目管理、素材库、成品预览、控制台
业务服务层：用户、权限、订单、算力、项目管理、模板管理
任务调度层：异步任务队列、AI 任务分发、超时重试、限流
AI 中台层：多模型接入、请求封装、参数统一、结果回调
数据存储层：数据库 + 对象存储 + 缓存

六、核心开发难点 & 解决方案
无限画布 + 大量节点卡顿
方案：节点虚拟渲染、画布分片加载、防抖缩放
AI 异步生成状态同步
方案：WebSocket 长连接 + 任务队列，实时推送进度
大视频前端处理卡顿
方案：ffmpeg.wasm 轻量化处理、服务端合成兜底
多模型参数不统一
方案：AI 中台统一入参出参，前端只调统一接口
